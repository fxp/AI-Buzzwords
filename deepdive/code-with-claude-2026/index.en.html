1.2 · Claude Managed Agents Three Major New Features

Beta header

A Lead Agent decomposes and delegates tasks to sub-agents, each with its own model, prompts, tools, and MCP servers. All agents run in the same container and file system, with results aggregated back to the Lead Agent's global context.

Overlaps conceptually with LangGraph and CrewAI, but Anthropic makes it a managed service with limits and observability. The 1-layer depth limit is a key design choice—forced flatness over the risk of runaway deep recursion. Customer case: Netflix platform team uses it to analyze hundreds of build logs in parallel.

PUBLIC BETA Outcomes · Results-Driven

Paradigm shift: from "imperative prompting" to "rubric-based definition"—developers write markdown rubrics describing "what success looks like," and an independent Grader Agent evaluates output in a fully isolated context window, preventing reasoning path contamination.

Default max 3 iterations, upper limit 20. Anthropic internal testing showed up to 10 percentage points improvement (.docx generation +8.4%, .pptx +10.1%). Medical document review company Wisedocs reduced review time by 50%.

Essentially a "productized Ralph loop"—upgrading the community's hand-written "loop→judge→retry" pattern to a first-class API citizen, making agent work verifiable rather than "run it and hope for the best."

RESEARCH PREVIEW Dreaming · Async Memory Introspection

Essence: an async, scheduled offline introspection process that does not modify model weights. Scans existing Memory Store + up to 100 historical sessions, distills patterns, cleans duplicate entries, and generates plain-text notes and "playbooks."

Supported models: claude-opus-4-7 and claude-sonnet-4-6. Requires dual beta headers: managed-agents-2026-04-01 + dreaming-2026-04-21.

Customer case: legal AI company Harvey saw ~6× improvement in task completion rate—the most-cited customer data at the conference.

Dreaming's real value: providing an "experience consolidation" middle layer for long-horizon agents, between RAG and fine-tuning. It doesn't learn new capabilities, only organizes old notes—media packaging it as "AI learning from mistakes" is overmarketing. Security warning: Pluto Security noted that prompt-injected "false memories" can persist across sessions via Memory Store.

1.3 · Claude Code Full-Stack Evolution

Boris Cherny's assessment cuts to the heart: "Going forward, a massive amount of code will be written asynchronously." Claude Code's product boundary at this conference expanded from a CLI tool to a continuously running layer spanning CLI, IDE, desktop GUI, web, mobile, CI integration, and scheduled tasks.

Fundamental difference between Routines and GitHub Actions: Routines run on Anthropic cloud; GitHub Actions call Claude from the user's own CI/CD environment. The former doesn't require a local machine and is the destination for judgment-intensive work; the latter is the destination for deterministic CI.
